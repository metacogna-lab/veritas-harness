# UNIFY_CLI — Unified Ingest CLI & App Placeholder

## Context

The current `bun run ingest` delegates to `src/cli.ts ingest` with minimal flags (`--input`, `--slug`, `--dry-run`). It has no TTY detection, no headless/agentic mode, no directory-batch processing, and no interactive interview for human operators. The ingest pathway must be promoted to a **standalone, dual-mode CLI** that serves both human operators (interactive interview) and autonomous agents (headless JSON output) without ambiguity. A UI is planned at `app/`; a placeholder must exist now so future work has a stable mount point.

## Goals

1. Standalone ingest CLI at `src/ingest/cli.ts` — the new `bun run ingest` entry point
2. Dual-mode: TTY-interactive (human) vs headless (agent/CI) via `process.stdout.isTTY`
3. Full flag set via `util.parseArgs` (native, zero deps)
4. Directory-batch mode (`--dir`) — scan, stage symlinks, compile plan, run Dogma Gate
5. JSON output mode (`--json`) — suppress all prose, emit structured result to stdout
6. Dogma Gate wired at end: `eval` → exit 0 on pass, cleanup + exit 1 on fail
7. App placeholder at repo root `app/` with README and minimal `package.json`

## Flag Specification

```
Usage: bun run ingest [options]

Options:
  -i, --interactive      Force interactive interview mode
  -s, --slug <string>    Mission identifier (required in headless mode)
  -o, --objective <str>  Mission objective (required in headless mode)
  -t, --target <path>    Scope boundary / target path
  -f, --files <paths>    Comma-separated list of .md/.pdf files to stage
  -d, --dir <path>       Batch-scan a directory; stage all .md/.pdf/.txt files
      --json             Suppress prose; emit { ok, planPath, slug, error? } to stdout
  -h, --help             Print this help
```

## Implementation Plan

### 1. `src/ingest/cli.ts` — Standalone Ingest CLI

New file. Replaces the `ingest` verb in `src/cli.ts`.

**Responsibilities:**
- Parse flags with `util.parseArgs` (types: `string` for slug/objective/target/files/dir, `boolean` for interactive/json/help)
- TTY detection: `const isInteractive = process.stdout.isTTY && !flags.json` — if not TTY and `--interactive` not forced and required flags missing → `stderr` error + `exit(1)`
- Route: `--dir` → `dirIngest()`, `--files` → `fileStageIngest()`, `--interactive` / bare → `interviewIngest()`
- After plan written: run Dogma Gate inline (`evalPlanWithConfig(plan)`) — pass → exit 0, fail → delete plan file + exit 1
- `--json` mode: suppress all `console.log`/`print()` before the final JSON line; always emit `JSON.stringify({ok, planPath, slug, error})` to stdout

### 2. `src/ingest/dir-scanner.ts` — Directory Batch Mode

New file.

**Responsibilities:**
- Accept `{ dirPath, slug, harnessRoot }` 
- `walkDir(dirPath)` — `readdirSync` recursive, filter `.md`, `.pdf`, `.txt`
- `stageFiles(files, slug, harnessRoot)` — create `research/raw/<slug>/`, idempotent `symlinkSync` (catch EEXIST → skip, catch other → throw)
- Build a synthetic `NEW.md` buffer in memory: frontmatter with `slug`, `title = slug`, `sources = [staged symlink paths]`, body section `## research question\n<auto-generated from slug>`
- Call `runIngest({ inputPath: syntheticPath, slug, harnessRoot })` on the written temp file (write to `research/raw/<slug>/AUTO_NEW.md`, delete after)
- Return `IngestResult`

### 3. `src/ingest/interview.ts` — Interactive Interview

New file.

**Responsibilities:**
- Use Node.js `readline/promises` (zero deps, available in Bun)
- Prompt sequence: slug → objective → target → (optional) context directory?
- Validate each field inline (non-empty, valid path if target/dir)
- Build `IngestOptions` from answers and call `runIngest()`
- Honour `--slug`/`--objective`/`--target` pre-fills (skip those prompts if already provided)

### 4. `src/ingest/ingest.ts` — Minor Extension

Extend `IngestOptions` with `syntheticContent?: string` so `dirScanner` can pass a pre-built NEW.md body without writing a temp file.

```ts
// If syntheticContent is set, skip readFileSync and use it directly
const raw = opts.syntheticContent ?? readFileSync(inputPath, "utf8");
```

## Data Flow

```
Human:  bun run ingest [-i]
          └─ interview.ts → answers → runIngest() → plan.json → evalPlanWithConfig() → exit 0/1

Agent:  bun run ingest --slug X --objective "..." --target /path [--json]
          └─ headless path → runIngest() → plan.json → evalPlanWithConfig() → JSON output → exit 0/1

Batch:  bun run ingest --slug X --dir /path/to/docs [--json]
          └─ dir-scanner.ts → symlinks → synthetic NEW.md → runIngest() → evalPlanWithConfig() → exit 0/1
```
