# Static Deployment Plan

Evaluated: 2026-07-12. Runtime: Bun + Node 18 + git. No native C++ modules. Fully headless CLI.

---

## Runtime Requirements

| Requirement | Detail |
|-------------|--------|
| **Bun** | Latest stable (pinned via `bun.lock`) |
| **Node.js** | 18+ (on PATH; checked by `bun run doctor`) |
| **Git** | On PATH (used by mission tooling) |
| **ANTHROPIC_API_KEY** | Required for default provider |
| **HARNESS_PROVIDER** | Optional — switch provider (anthropic, ollama, openai, openrouter, claude-code, codex) |
| **HARNESS_MODEL** | Optional — override model for active provider |
| **VERITAS_RUNS_DIR** | Optional — mission store location (default: `.veritas/runs`) |
| **OPENAI_API_KEY** | Required only if HARNESS_PROVIDER=openai |
| **OPENROUTER_API_KEY** | Required only if HARNESS_PROVIDER=openrouter |

### Native module surface

None. `esbuild` ships pre-built multi-arch JS binaries (included for Linux x64, ARM64, macOS). No Python, no C++ toolchain, no node-gyp. Clean JavaScript/TypeScript stack throughout.

### Interaction model

The CLI is **fully headless** — no interactive stdin, no readline. All verbs (`start`, `status`, `report`, `loadouts`, `ingest`, `eval`, `digest`, `rsi`) stream events to stdout and exit with a numeric code. This makes both Docker and Modal straightforward.

## Persistent State

| Directory | Contents | Persistence |
|-----------|----------|-------------|
| `.veritas/runs/` | Mission snapshots (`<id>.json`) | **Persist** — needed for `status` and `report` |
| `missions/` | Ingested research plans (`<slug>/research-plan.json`) | **Persist** — input to `start --plan` |
| `resources/experience/` | Per-mission RSI input (transcript.jsonl, findings.jsonl) | **Persist** — feeds RSI outer loop |
| `resources/lessons.json` | Lessons delta store | **Persist** — RSI feedback state |
| `resources/summary/` | Generated source digests | **Ephemeral** — re-generated by `bun run dev digest` |
| `node_modules/` | Dependencies | **Ephemeral** — rebuilt by `bun install` |
| `build/` | TypeScript output | **Ephemeral** — rebuilt by `bun run build` |

Volume-mount the three **Persist** directories; everything else can be rebuilt.

## Approach A — Docker (Primary Recommendation)

**Best for:** local development, CI, long-running audits, Kubernetes/ECS deployments.

### Files to create

### `Dockerfile`

```dockerfile
# Stage 1: install dependencies (layer-cached separately from source)
FROM oven/bun:latest AS deps
WORKDIR /app
COPY package.json bun.lock ./
RUN bun install --frozen-lockfile

# Stage 2: runtime image
FROM oven/bun:latest
# git is required by mission tooling
RUN apt-get update && apt-get install -y --no-install-recommends git \
    && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

# Secrets NEVER baked into the image — always injected at runtime
# via --env-file or orchestrator secrets

HEALTHCHECK --interval=30s --timeout=10s --start-period=15s --retries=3 \
    CMD bun run doctor || exit 1

# Persistent state lives outside the image layer
VOLUME ["/app/.veritas", "/app/missions", "/app/resources/experience"]

# Default: list loadouts (safe no-op; override CMD to run a mission)
ENTRYPOINT ["bun", "run", "dev"]
CMD ["loadouts"]
```

## Approach B — Modal (Serverless / Scheduled Runs)

**Best for:** scheduled research runs (daily/weekly), cost-optimized batch missions, no infrastructure to maintain.

**Tradeoff vs Docker:** Modal has a 5–10 s cold start; no long-running process cost. Docker has consistent warm latency but requires a host. For missions that run once and produce a report, Modal wins on cost and ops overhead.

### Files to create

```
harness/veritas-example/
  modal_app.py
  modal_volume_setup.py
```

### `modal_app.py`

```python
"""
Veritas research harness on Modal.

Deploy:  modal deploy modal_app.py
Run:     modal run modal_app.py::run_mission --objective "..." --target "..."
Schedule: modal deploy modal_app.py  (cron configured below)
"""
import subprocess
import os
import modal

app = modal.App("veritas-example")

# Build image: Debian slim + Bun + project dependencies
image = (
    modal.Image.debian_slim(python_version="3.12")
    .run_commands(
        # Install Bun
        "apt-get update && apt-get install -y --no-install-recommends git curl unzip && "
        "curl -fsSL https://bun.sh/install | bash",
        "echo 'export PATH=$HOME/.bun/bin:$PATH' >> /etc/environment",
    )
    .copy_local_dir(
        # Adjust path relative to where you run modal deploy from
        local_path=".",
        remote_path="/app",
    )
    .run_commands(
        "cd /app && /root/.bun/bin/bun install --frozen-lockfile"
    )
)

# Persistent volume for mission state (shared across runs)
runs_volume = modal.Volume.from_name("veritas-runs", create_if_missing=True)
experience_volume = modal.Volume.from_name("veritas-experience", create_if_missing=True)

BUN = "/root/.bun/bin/bun"


@app.function(
    image=image,
    secrets=[modal.Secret.from_name("anthropic-api")],  # key: ANTHROPIC_API_KEY
    volumes={
        "/app/.veritas/runs": runs_volume,
        "/app/resources/experience": experience_volume,
    },
    timeout=3600,  # 1 hour ceiling; increase for very long missions
)
def run_mission(objective: str, target: str, loadout: str = "research") -> dict:
    env = {**os.environ, "VERITAS_RUNS_DIR": "/app/.veritas/runs"}
    result = subprocess.run(
        [BUN, "run", "dev", "start", objective, "--target", target, "--loadout", loadout],
        cwd="/app",
        env=env,
        capture_output=True,
        text=True,
        timeout=3500,
    )
    return {
        "stdout": result.stdout,
        "stderr": result.stderr,
        "returncode": result.returncode,
    }


# Optional: scheduled daily research run
@app.function(
    image=image,
    secrets=[modal.Secret.from_name("anthropic-api")],
    volumes={
        "/app/.veritas/runs": runs_volume,
        "/app/resources/experience": experience_volume,
    },
    schedule=modal.Cron("0 6 * * *"),  # 06:00 UTC daily
    timeout=3600,
)
def daily_rsi():
    """Run the RSI outer loop daily to propose harness improvements."""
    env = {**os.environ, "VERITAS_RUNS_DIR": "/app/.veritas/runs"}
    subprocess.run(
        [BUN, "run", "dev", "rsi"],
        cwd="/app",
        env=env,
        check=True,
    )
```

## Comparison

| Criterion | Docker | Modal |
|-----------|--------|-------|
| Setup complexity | Low (one Dockerfile) | Medium (Python wrapper + secrets) |
| Cold start latency | <2 s (cached image) | 5–10 s first run, <2 s thereafter |
| Persistent state | Named volumes (local) or EFS/EBS | Modal volumes (cloud) |
| Cost model | Compute + storage always-on | Pay per execution second |
| Scheduling | External (cron, Kubernetes CronJob) | Built-in (`modal.Cron`) |
| Long-running missions | Ideal | Requires timeout ≤ 1 h (configurable) |
| Multi-provider CLI | Include binaries in image | Same (in Modal image build) |
| Operational overhead | Host/cluster management | Zero (Modal manages) |
| Recommended for | Dev, CI, long audits, ECS/K8s | Scheduled research, cost-optimized batch |

**Recommendation:** Docker-first. Add Modal as the scheduled-run overlay for veritas-example once Docker is stable. No harness code changes required for either approach.

## Security Notes

1. **Secrets never in images.** `ANTHROPIC_API_KEY` is always runtime-injected (`--env-file`, orchestrator secrets, Modal secrets store). The `.dockerignore` excludes `.env*` and `src/config/local.json`.
2. **Health check.** `bun run doctor` verifies key presence and redaction before any mission starts. Use it as the container HEALTHCHECK and as a pre-flight in CI.
3. **Scope gate holds.** The scope gate (`src/safety/scope.ts`) enforces mission-declared paths/hosts regardless of runtime environment. Containerization does not weaken it.
4. **State volume permissions.** Run the container as a non-root user in production; volume mount paths need write permission for that UID.
