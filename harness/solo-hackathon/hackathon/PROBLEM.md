# PROBLEM — Solo Hackathon Kit

## One-liner

A solo builder needs a complete, sitting-sized process kit *before* coding — not invented mid-sprint.

## Why it matters

Without a framed problem, constraints, success bar, MVP cut, and demo package, a one-person hackathon collapses into either thrash (no scope) or theatre (process without a shippable artefact).

## The problem this pack solves

**Produce every operational file for a single-person hackathon** so the facilitator loadout can hand the builder a kit and step back:

1. Clear problem statement (this file)
2. Hard constraints (time / tools / out-of-scope)
3. Measurable success criteria
4. MVP vertical slice
5. Timeline for one sitting
6. Build checklist
7. Demo package outline
8. Agenda + decisions log

## Non-goals (of the pack itself)

- Not a multi-team event toolkit
- Not a judging platform
- Not a replacement for the Veritas agent loop — the harness *runs* missions; these files *shape* them

## Target user

One builder, one sitting, one shippable artefact — assisted by the `solo-hackathon` loadout (facilitator → scout → builder).
